NOTABLE DESIGN DECISIONS
================================================================================

1. JSON Fields (login_ids, additional_ips)
   - Stored as json() column type, cast to array via model $casts
   - Controllers (both API + Web) decode the validated JSON string to array
     before passing to create/update, preventing double-encoding by the
     array cast mutator
   - Not included in search (MySQL JSON text search is unreliable across
     different drivers/configurations)

2. cloudflare_status
   - String(20) column + app-level in: validation (enabled, disabled, unknown)
   - NOT a DB enum — allows adding new values in the future without a migration
   - Purely manual entry; no Cloudflare API integration (Phase C constraint)

3. IP Address Columns (domain_ip, mail_domain_ip, cpanel_ip)
   - String(45) for IPv6 compatibility (same pattern as existing ip_address)
   - Simple text fields — no IP validation, accommodating internal/hostname values

4. Email Field (service_providers)
   - Uses string|email validation (not email:rfc,dns) — avoids DNS lookup
     failures on internal/private domains

5. Export/Import
   - Columns auto-mapped via $fillable — no ImportController changes needed
   - Export controller column arrays updated to include new fields

6. Audit Logging
   - All new fields in $fillable are automatically tracked by Spatie
     Activitylog via logFillable() in the model trait
