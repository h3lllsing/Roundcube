===========================================================

Date: 2026-06-26
Status: COMPLETE

1. SEARCH ENGINE
----------------

File: app/Services/GlobalSearchService.php (new)
- 15 searchable modules configured (Domains, Hosting, VPS, VoIP, Domain Emails,
  Other Services, Service Providers, Renewals, Assets, Tasks, Vault, Notes,
  Features, Modules, Users, SMTP Profiles)
- Multi-column LIKE '%term%' per module (no raw MySQL)
- Ownership scoping:
  - sa_only (Features, Modules, Users, SMTP Profiles) — hidden from non-admin
  - user (Notes) — only own records
  - user_or_module (Domains, Hosting, VPS, etc.) — own records or module-accessible
  - task (Tasks) — in accessible modules or assigned to user
- Relevance ordering: Exact Match (3) → Starts With (2) → Contains (1)
- Max 5 results per module, max 50 total
- 6 filters: All, Services, Assets, Tasks, Vault, Users
- searchForApi() returns indexed array groups for JSON API
- SQLite-safe, no Redis, no Scout, no migrations

2. SEARCH HELPER
----------------

File: app/Helpers/SearchHelper.php (new)
- static highlight(): wraps matching portions in <mark class="bg-amber-200...">
- Case-insensitive matching, preserves original case
- Returns raw HTML string for use with {!! !!} in Blade

3. API SEARCH CONTROLLER
------------------------

File: app/Http/Controllers/Api/SearchController.php (refactored)
- Delegates to GlobalSearchService
- 60s per-user cache (keyed by user_id + query hash + filter + limit)
- Throttled (search limiter + api limiter)
- Accepts: q (required, min 2 chars), filter (default: all), limit (default: 5, max: 20)
- Returns: { data: [{ key, label, url, items }] }
- Swagger/OpenAPI annotations preserved
