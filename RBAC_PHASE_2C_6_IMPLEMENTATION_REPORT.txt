RBAC Phase 2C-6: Enterprise Permission Inspector
=================================================
Date: 2026-06-25

1. OBJECTIVE
   Transform the User Details page into an Enterprise Permission Inspector for
   Super Admin, displaying ALL modules with effective permissions, summary
   cards, and visual source badges.

2. FILES CHANGED
   - app/Http/Controllers/Web/UserController.php  (modified — show method)
   - resources/views/users/show.blade.php           (modified — full restructure)
   - tests/Feature/RbacPhase2C6Test.php             (new)

3. FILES NOT CHANGED
   - app/Traits/HasModulePermissions.php  (RBAC logic untouched)
   - app/Http/Controllers/Web/AuthController.php
   - resources/views/auth/my-permissions.blade.php
   - Any routes, migrations, API controllers, or models
   - Server-side guards, sidebar, dashboard, CRUD views

4. CONTROLLER CHANGES
   UserController@show now:
   - Loads ALL modules (Module::with('feature')->orderBy('name')->get())
     instead of only modules with role permissions.
   - Computes 6 summary statistics from the in-memory modulePermissions
     collection (no extra DB queries beyond existing getEffectiveModulePermissions).
   - Detects if inspected user is super-admin ($inspectedIsSuperAdmin flag).
   - Fetches last login timestamp from LoginAudit table.
   - Passes $summary, $inspectedIsSuperAdmin, and $lastLogin to the view.

5. SUMMARY CARDS ADDED (6 cards)
   - Roles              — indigo   — count of user's assigned roles
   - Accessible Modules — emerald  — modules where can_read effective = true
   - Denied Modules     — rose     — modules where can_read effective = false
   - Overrides          — violet   — count of non-null user overrides across all modules
   - Allowed Permissions — emerald — count of all effective = true permissions
   - Denied Permissions — rose     — count of all effective = false permissions

   Uses existing <x-stat-card> component for consistent dashboard styling.

6. PERMISSION MATRIX CHANGES
   - Now shows ALL modules, including those without any role permissions.
   - Table headers expanded to full names: Read, Create, Update, Delete,
     Approve, Export, Reveal, Import.
   - Each permission cell shows:
       ✓ (green) if effective = true
       ✕ (red)   if effective = false
     Below it, a source badge:
       - Role            (blue text)   — permission from role
       - Override Allow  (purple text) — user override granting permission
       - Override Deny   (purple text) — user override denying permission
       - None            (gray text)   — no role or override set
       - Super Admin     (green text)  — inspected user is super-admin

7. BADGE SYSTEM
   Source labels are rendered as inline text spans with color classes:
   - Role:            text-blue-600 dark:text-blue-400
   - Override Allow:  text-purple-600 dark:text-purple-400
   - Override Deny:   text-purple-600 dark:text-purple-400
   - None:            text-gray-400 dark:text-gray-500
   - Super Admin:     text-green-600 dark:text-green-400

8. SUPER ADMIN BANNER
   When the inspected user is a Super Admin, a green banner displays:
   "This user has unrestricted access through the Super Admin role."

9. USER SUMMARY HEADER
   Compact enterprise layout with ID, Email, Roles, Status (Active/Suspended
   badge), and Last Login (when available) in a 4-column grid.
