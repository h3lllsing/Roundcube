                    search (asset_tag, serial_number, department, notes)

    - store()     — delegates to AssetService
    - show()      — loads full relations, ownership check
    - update()    — ownership check
    - destroy()   — ownership check, soft delete

12. ASSET ASSIGNMENT WORKFLOW
================================================================================

  ┌──────────────────────────────────────────────────────────────────────────┐
  │                                                                          │
  │  available  ---(assign)--->  assigned  ---(return)--->  available        │
  │      |                            |                                      │
  │      +---> lost                   +---> lost                             │
  │      +---> decommissioned         +---> decommissioned                   │
  │                                                                          │
  │  lost, decommissioned = terminal (no transitions out)                    │
  │                                                                          │
  └──────────────────────────────────────────────────────────────────────────┘

  Assign action:
    - Validates: assigned_to (required), department, issue_date,
      expected_return_at, assignment_reason (select: New Employee/Replacement/
      Temporary/Loan/Other), note
    - Creates asset_assignments row with returned_at=NULL
    - Updates asset: assigned_to=user, status='assigned', issue_date=today,
      return_date=NULL
    - Transaction-wrapped (rolls back if either step fails)

  Return action:
    - Validates: condition_on_return (new/good/fair/poor/damaged), note
    - Closes active assignment: sets returned_at=now()
    - Updates asset: assigned_to=NULL, status='available', return_date=today
    - Transaction-wrapped

  Additional Attachments:

    - Attachments are polymorphic (notable morph) via Attachment model
    - Uploaded via existing Attachments UI (/attachments/create) with
      notable_type = App\Models\Asset and notable_id = asset ID
    - Displayed on show view as a grid of downloadable links
    - Attachment count shown on show page header
