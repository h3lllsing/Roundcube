================================================================================
CLONE INTEGRATION
===============================================================??===============

During user creation, clone is an optional step. When selected:

  1. Source user roles are copied (super-admin role is filtered out)
  2. Source user overrides are copied (all UserModulePermission rows)
  3. Source user status (suspended_at) is optionally copied
  4. Clone role_handling strategy determines final role set:
     - use_cloned (default): only cloned roles
     - replace: only manually selected roles
     - merge: union of cloned + selected roles
  5. Password is NEVER copied — always required from form input

================================================================================
ROLE PREVIEW IMPLEMENTATION
================================================================================

The Role Preview is implemented as a READ ONLY, INFORMATIONAL component:

  Server side (create()):
    - Queries ModuleRolePermission grouped by role_id
    - Queries RoleTemplate keyed by slug
    - For each role, computes: modules_count, presence of each permission
      (can_read, can_create, can_update, can_delete, can_reveal, can_export)
    - If a template slug matches the role slug, attaches template name

  Client side (view):
    - Each role has a hidden preview card (class="role-preview hidden")
    - When role checkbox is checked, corresponding preview card is shown
    - JS: document.querySelectorAll('.role-checkbox') + change event
    - Summary section updates role names dynamically

  Boundaries:
    - Does NOT call applyTemplateToRole()
    - Does NOT write to ModuleRolePermission
    - Does NOT create/update templates
    - Is purely informational
