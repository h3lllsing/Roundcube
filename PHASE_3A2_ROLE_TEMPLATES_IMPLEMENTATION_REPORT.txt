APPLY FLOW
----------
1. Super Admin navigates to template Show page
2. Selects target role from dropdown
3. Clicks "Preview & Apply" (GET) -> sees diff preview
   - Green: modules to be added
   - Amber: modules to be overwritten (old vs new values shown)
   - Gray: unchanged module count
   - Red warning if template is dangerous
4. If satisfied, checks confirmation (plus dangerous confirmation if needed)
5. Clicks "Confirm & Apply" (POST)
6. System:
   a. Validates dangerous confirmation (if applicable)
   b. Wraps in DB::transaction()
   c. For each module in permissions_json:
      - If existing ModuleRolePermission matches -> skip (unchanged count)
      - If existing differs -> update (changed count)
      - If no existing -> create (added count)
   d. Logs activity with event 'template_applied' including template/role/changed/added/unchanged counts
   e. Redirects to template show page with success message
