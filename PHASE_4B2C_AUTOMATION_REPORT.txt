IMPLEMENTATION
================================================================================

1. Artisan Command — app/Console/Commands/SendEmailReminders.php

   Signature: renewals:send-email-reminders {--limit=}

   Behavior:
   - Counts eligible trackers (notifications enabled + active/pending_renewal)
   - Calls RenewalNotificationService::sendReminders($limit)
   - Queries failed notifications created during the run
   - Displays summary table: Processed | Sent | Skipped | Failed
   - Logs summary via logger() (no SMTP passwords)
   - Returns Command::SUCCESS

   Options:
   --limit=N   Stop after N emails sent (unlimited by default)

2. Scheduler — routes/console.php

   Schedule::command('renewals:send-email-reminders')->dailyAt('02:00');

   Shared hosting compatible (Laravel scheduler only, no system cron).

3. Processing

   - Uses RenewalNotificationService::sendReminders()
   - trigger_source = 'cron' for all automatic sends
   - preventDuplicate() respected (dedup by tracker+day+recipient+source)
   - Each email creates a history row
   - Exceptions per recipient caught — next recipient/tracker continues
   - Rate limiting via --limit option

4. Selection Criteria (inside sendReminders())

   - email_notifications_enabled = true
   - status IN (active, pending_renewal)
   - Reminder due today (matching day in notify_days_before)
   - Not already sent (preventDuplicate check)

   Inactive SMTP profiles fall back to default mailer — nothing skipped.
