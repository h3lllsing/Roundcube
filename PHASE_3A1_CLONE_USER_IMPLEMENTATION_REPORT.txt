CLONE BEHAVIOR SUMMARY
----------------------
- Only Super Admin can clone users
- Password is ALWAYS required (never copied)
- Roles: can be copied (optional checkbox)
- Overrides: can be copied (optional checkbox)
- Status (suspended_at): can be copied (optional checkbox)
- Cloning a Super Admin requires explicit confirmation checkbox
- Source user is never modified
- Activity logged as 'cloned_user' type
- New user gets fresh avatar, remember_token, timestamps
- Owned records, sessions, MFA, API tokens NOT transferred

KEY IMPLEMENTATION DETAILS
--------------------------
- suspended_at bypasses $fillable via forceFill() after User::create()
  (same pattern as existing UserController store/update methods)
- DB::transaction() ensures atomicity - failed clones roll back completely
- CloneUserRequest filters non-super-admin attributes in prepareForValidation
- User is only considered super admin if is_super_admin() returns true
  (not just has super-admin role assigned)
