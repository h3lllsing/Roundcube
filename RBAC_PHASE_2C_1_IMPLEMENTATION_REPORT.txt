4. BUTTON VISIBILITY LOGIC
--------------------------------------------------------------------------------

  Create button:
    Visible if $canCreate = true.
    $canCreate = super-admin OR (module exists AND canOnModule(module, 'create')).

  Export CSV button:
    Visible if $canExport = true.
    $canExport = super-admin OR (module exists AND canOnModule(module, 'export')).

  Bulk action "Delete":
    Appears in dropdown only if $canBulkDelete = true.
    $canBulkDelete = super-admin OR (module exists AND canOnModule(module, 'delete')).

  Bulk action "Restore":
    Appears only if $canBulkRestore = true (super-admin only).

  Bulk action "Force delete":
    Appears only if $canBulkForceDelete = true (super-admin only).

  Bulk action "Update status":
    Always included for service-type modules. Not included for vault.

  If the resulting $bulkActions array is non-empty (always has at least
  'update-status' for service modules), the bulk-actions component renders
  with those options. If empty (impossible for current modules but handled
  correctly), the component would still render with an empty action list.

  User override behaviour:
    All permission checks use canOnModule(), which resolves:
      1. UserModulePermission override (if non-null) → return override value
      2. Otherwise → return true if any role grants the permission
    This means user overrides are automatically respected without any
    additional code.

8. SECURITY
--------------------------------------------------------------------------------

  - UI-only enforcement: This phase adds view-level hiding. Server-side
    403 guards remain the authoritative enforcement. A user who sees a
    hidden button and manually navigates to the create URL will still
    receive a 403.

  - No privilege escalation: The @if checks only hide UI elements. They
    never grant access.

  - User overrides correctly handled: UserModulePermission overrides flow
    through canOnModule() and are respected by all button visibility checks.

  - Super-admin always sees everything: Super-admin bypasses all permission
    checks via $isSuperAdmin = true.
